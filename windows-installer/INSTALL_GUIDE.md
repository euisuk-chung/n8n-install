# n8n 설치 가이드 (Windows)

비개발자를 위한 n8n 설치 매뉴얼입니다. 명령 프롬프트나 PowerShell 사용 경험이 없어도 따라할 수 있습니다.

## 시스템 요구사항

- Windows 10 (1809 이상) 또는 Windows 11
- 약 1GB 의 여유 디스크 공간
- 인터넷 연결 (첫 실행 시 n8n 본체 다운로드용)

## 설치하기

1. **인스톨러 다운로드**
   - `n8n-installer-x.x.x.exe` 파일을 다운로드합니다.

2. **더블클릭으로 설치 시작**
   - 다운로드한 파일을 더블클릭하면 한국어 설치 마법사가 시작됩니다.
   - "Windows의 PC 보호" 경고가 나오면 → **추가 정보** → **실행** 을 누르세요.
     (공식 서명이 아직 적용되지 않아 나타나는 정상적인 경고입니다.)

3. **약관 동의 및 옵션 선택**
   - 라이선스 약관 동의
   - 설치 폴더 (기본값 권장: `C:\Users\<사용자>\AppData\Local\Programs\n8n`)
   - 추가 옵션:
     - ✅ **Windows 시작 시 자동 실행** (권장)
     - ✅ **바탕화면 바로가기 만들기**
     - ✅ **지금 n8n 시작**

4. **설치 진행**
   - 진행률 바가 100% 가 되면 완료입니다.

## 처음 실행

설치 직후 화면 오른쪽 아래 트레이 영역(시계 근처)에 **n8n 아이콘**이 나타납니다.

처음 실행 시 약 **1~2 분** 동안 n8n 본체를 다운로드합니다. 진행 창이 자동으로 표시됩니다.

다운로드가 끝나면 **기본 브라우저**가 자동으로 열리며 `http://localhost:5678` 의 n8n 첫 화면이 나타납니다.

n8n 초기 설정 안내(이메일, 관리자 비밀번호 등)를 따라 진행하세요.

## 일상 사용법

트레이 아이콘을 **우클릭**하면 메뉴가 나타납니다:

| 메뉴 | 설명 |
|------|------|
| n8n 열기 | 브라우저에서 n8n 화면 열기 |
| n8n 시작 / 중지 | n8n 서버 시작 또는 중지 |
| 로그 보기 | 문제 발생 시 로그 확인 (메모장에서 열림) |
| n8n 업데이트 | 최신 버전으로 업데이트 |
| 데이터 폴더 열기 | 워크플로/자격증명 저장 위치 확인 |
| Windows 시작 시 자동 실행 | 부팅 시 자동 실행 토글 |
| 정보 | 버전 정보 |
| 종료 | n8n 종료 |

좌클릭(또는 더블클릭): 브라우저로 n8n 화면을 빠르게 엽니다.

## 데이터 저장 위치

워크플로, 자격증명, 실행 이력 등 모든 사용자 데이터는 다음 폴더에 저장됩니다:

```
C:\Users\<사용자>\.n8n
```

- 백업 시 이 폴더 전체를 복사하세요.
- 다른 PC 로 이주 시: 새 PC 에 n8n 설치 후 이 폴더를 동일 경로에 복사

## 업데이트

트레이 아이콘 우클릭 → **n8n 업데이트** 클릭. 약 1~2분 소요됩니다.

업데이트 후에는 n8n 이 자동으로 재시작됩니다.

## 제거

- **제어판 → 프로그램 → n8n → 제거**
- 또는 시작 메뉴에서 "n8n 제거" 검색

⚠️ 사용자 데이터(`%USERPROFILE%\.n8n`)는 **보존됩니다**. 완전 삭제를 원하시면 제거 마법사의 "사용자 데이터도 삭제" 체크박스를 선택하세요.

## 자주 묻는 질문

### Q. "이 앱은 PC를 보호하기 위해 차단되었습니다" 가 떠요

A. 인스톨러가 아직 코드 서명되지 않아서 발생합니다. **추가 정보** → **실행** 을 누르세요. n8n 은 오픈소스이므로 GitHub 에서 원본 코드를 확인할 수 있습니다.

### Q. 브라우저가 자동으로 열리지 않아요

A. 5~10초 정도 기다려보세요. 그래도 안 열리면 트레이 아이콘 우클릭 → **n8n 열기** 를 선택하세요. 5678 포트가 다른 프로그램에 사용 중이면 5679, 5680… 으로 자동 변경되며 툴팁에 표시됩니다.

### Q. 회사 PC 에서 첫 실행 시 다운로드가 실패해요

A. 회사 방화벽/프록시가 npm 레지스트리(`registry.npmjs.org`)를 차단했을 가능성이 큽니다. IT 부서에 다음 도메인 허용을 요청하세요:

- `registry.npmjs.org` (HTTPS 443)
- `nodejs.org` (업데이트 시)

프록시 환경이라면 트레이 아이콘 우클릭 → **데이터 폴더 열기** 에서 같은 폴더에 `.npmrc` 파일을 만들고:

```
proxy=http://<사내프록시>:<포트>
https-proxy=http://<사내프록시>:<포트>
registry=https://registry.npmjs.org/
```

### Q. n8n 이 시작하지 않아요

A. 트레이 우클릭 → **로그 보기** 로 오류 메시지를 확인하세요. 일반적으로:

- 5678 포트 충돌: 다른 프로그램(Skype 등) 종료 후 재시도
- Node.js 손상: 인스톨러 재실행으로 복구
- 데이터 폴더 권한 문제: `%USERPROFILE%\.n8n` 의 쓰기 권한 확인

### Q. 백업/복원은 어떻게 하나요

A. `%USERPROFILE%\.n8n` 폴더를 압축해서 보관하세요. 복원 시 동일 경로에 압축 해제 후 트레이에서 "n8n 시작".
